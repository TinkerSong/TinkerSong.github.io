# 备份

备份问题一致困扰我很久，其需求来自于自己一个错误操作，导致自己的文章和配置丢失无法找回，以及我平时是在笔记本上撰写文章，但是回家以后我是在台式机上写文章的，2 端操作及其频繁，所以就在想可否把自己的原始代码也上传到 github 上，这样我就可以通过 git 来进行更新备份，进过尝试发现这样的操作是完全可行的。接下来就记录一下具体的操作方法，供参考。

首先，在 github 上创建 2 个分支，一个 hexo，一个 master

master 分支用于保存 Hexo 所部署的页面静态文件

hexo 分支用于保存页面的源代码文件。

这样一来既不影响我们部署静态页面，同时也能备份 hexo 源码实现版本管理，写完文档以后直接部署，上传代码，在其他电脑上只要安装 node 环境和插件，下载源码就可以接着用了，相对来说还是比较方便。
